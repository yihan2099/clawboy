# V1 Legacy Cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove all V1 legacy remnants (disputes, tasks_won, claiming, staking) from the codebase and database to achieve full V2 consistency.

**Architecture:** Database migration first (drop dead columns/functions), then update TypeScript types, then update frontend display, then fix tests. Each task produces a buildable state.

**Tech Stack:** Supabase (SQL), TypeScript, Next.js 16, Bun

**Spec:** `docs/superpowers/specs/2026-03-18-v1-cleanup-design.md`

### Task 1: Database migration — drop dead columns and functions

**Files:**
- Supabase database (via MCP tool)

- [ ] **Step 1: Drop dead columns from agents table**

Run SQL via Supabase MCP:
```sql
ALTER TABLE agents DROP COLUMN IF EXISTS tasks_won;
ALTER TABLE agents DROP COLUMN IF EXISTS disputes_won;
ALTER TABLE agents DROP COLUMN IF EXISTS disputes_lost;
```

- [ ] **Step 2: Drop dead columns from tasks table**

```sql
ALTER TABLE tasks DROP COLUMN IF EXISTS claimed_by;
ALTER TABLE tasks DROP COLUMN IF EXISTS claimed_at;
```

- [ ] **Step 3: Drop dead RPC functions**

```sql
DROP FUNCTION IF EXISTS increment_tasks_won(text);
DROP FUNCTION IF EXISTS mark_submission_winner(uuid, text);
```

- [ ] **Step 4: Update list_tasks_with_bounty_filter to remove claimed_by**

First check current function signature:
```sql
SELECT pg_get_functiondef(oid) FROM pg_proc WHERE proname = 'list_tasks_with_bounty_filter';
```

Then recreate it without `p_claimed_by` parameter and without `claimed_by`/`claimed_at` in the return type and SELECT.

- [ ] **Step 5: Update count_tasks_with_bounty_filter to remove claimed_by**

Same approach — remove `p_claimed_by` parameter and its WHERE clause.

- [ ] **Step 6: Verify no errors**

```sql

SELECT count(*) FROM tasks;
```
